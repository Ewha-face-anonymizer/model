# 우리아이만 실행 계획 (성능실험 + UX + 개발)

## 1. 목표와 원칙
- 목표: `우리 아이만 비블러, 나머지는 블러`를 안정적으로 제공한다.
- 핵심 우선순위:
1. 아동 얼굴 누락 최소화(안전)
2. 처리 속도/비용 균형(학생 예산)
3. 사용자가 쉽게 이해하고 바로 쓸 수 있는 UX
- 기본 원칙: 확신이 낮으면 비블러하지 않고 블러를 유지한다.

## 2. 성능 실험 계획 (2주)

## Week A: 기준선(Baseline) 측정
- 목적: 현재 파이프라인의 정확도/속도/비용 기준값 확보
- 실험 조건 고정:
  - 입력: 동일 영상 세트(실내/실외/단체/움직임)
  - 해상도: 720p 우선
  - 길이: 10초, 30초, 60초
  - 하드웨어: 동일 머신(CPU/GPU 분리 기록)
- 측정 지표:
  - 정확도: 누락률(FN), 오검출률(FP)
  - 속도: 총 처리시간, 평균 FPS, p95 지연
  - 비용: 요청당 평균 처리 자원(시간/메모리)
- 산출물:
  - Baseline 리포트 1개
  - 실패 케이스 샘플(영상 타임스탬프 포함)

## Week B: 개선안 A/B 테스트
- 후보 개선안:
1. `프레임 스킵`: 매 프레임 탐지 -> N프레임마다 탐지
2. `리사이즈`: 1080p -> 720p 입력 고정
3. `트래킹`: detect + track(탐지 횟수 절감)
4. `매칭 안정화`: 연속 프레임 투표(단일 프레임 오판 완화)
- 합격 기준(예시):
  - Baseline 대비 처리시간 20% 이상 개선
  - 아동 대상 누락률 악화 없음(또는 개선)
  - 메모리 급증/타임아웃 없음
- 산출물:
  - 최종 채택 설정값(config)
  - v0.1 운영 파라미터 표

## 3. 데이터/평가셋 구성 지침 (아동 특화)
- 등록 이미지:
  - 아이 1명당 5~10장
  - 정면/측면/표정/조명/마스크 유무 다양화
- 평가 영상:
  - 작은 얼굴(원거리), 가림(모자/손), 빠른 움직임 필수 포함
- 분리 원칙:
  - 등록용 이미지와 평가 영상 분리(데이터 누수 방지)
- 재등록 정책:
  - 3~6개월 주기 업데이트 권장

## 4. UX 설계 시 필수 고려사항

## 4.1 온보딩/등록 UX
- 첫 화면에서 기능을 한 문장으로 설명:
  - "등록한 우리 아이 얼굴만 남기고 자동 블러합니다."
- 등록 단계는 3단계로 단순화:
1. 로그인
2. 얼굴 사진 5장 등록
3. 동영상 업로드
- 등록 품질 가이드 제공:
  - 정면/밝은 사진 권장, 너무 작거나 흐린 사진 경고

## 4.2 처리/결과 UX
- 처리 상태를 명확히 표시:
  - 대기중 -> 처리중 -> 완료/실패
- 완료 후 즉시 다운로드 버튼 제공
- 실패 시 재시도 버튼 + 사람이 이해 가능한 에러 문구 제공

## 4.3 신뢰 UX(중요)
- "확신이 낮은 얼굴은 블러 유지" 정책을 사용자에게 명시
- 자동 삭제 시간(예: 24시간)과 남은 시간 표시
- "내 데이터 즉시 삭제" 버튼 상시 노출

## 5. 개발 시 고려사항 (엔지니어링 체크리스트)

## 5.1 파이프라인/모델
- 전처리 채널(BGR/RGB) 일관성 검증
- 검출/임베딩/매칭 로그를 단계별로 남기기
- 모델 버전과 임계값을 결과 메타데이터에 기록

## 5.2 백엔드/API
- 비동기 Job 구조:
  - 업로드 -> job_id 발급 -> 상태 조회 -> 결과 다운로드
- 파일 크기/길이/포맷 검증 선처리
- 타임아웃, 재시도, 실패 사유 코드 표준화

## 5.3 스토리지/삭제
- 원본 업로드와 결과 파일 TTL 강제(자동 삭제)
- 임베딩과 파일 저장소 분리
- 사용자별 접근 격리(권한 체크)

## 5.4 보안/개인정보
- 임베딩도 민감정보로 취급
- HTTPS, 저장 암호화, 서명 URL 짧은 만료
- 계정 탈퇴/삭제 요청 시 임베딩 포함 완전 삭제

## 5.5 운영/비용
- 동시 처리 수 제한(초기 1~2)
- 사용량 로깅(요청 수, 처리시간, 실패율)
- 무료 구간 제한(예: 30초, 720p)

## 6. 릴리즈 게이트 (v0.1)
- 정확도:
  - 아동 테스트셋에서 누락률 기준 통과
- 성능:
  - 30초/720p 처리시간 목표 통과
- 안정성:
  - 치명 오류율 기준 이하
- 신뢰:
  - 삭제/보안/고지 문구 반영 완료

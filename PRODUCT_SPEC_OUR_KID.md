# 우리아이만(가칭) 확장 스펙 v0.1

## 1) 문제 정의
- 학부모는 단체 영상(학예회, 운동회, 수업 영상)에서 `우리 아이만` 보이게 하고, 나머지는 익명화해서 공유하고 싶다.
- 기존 도구는 수동 편집이 어렵거나 시간이 오래 걸린다.

## 2) 제품 한 줄 정의
- `우리아이만`: 아이 얼굴을 등록해두면, 업로드한 동영상에서 등록된 아이만 남기고 나머지 얼굴은 자동 블러 처리해 주는 웹 서비스.

## 3) 타겟 사용자
- 1차: 초등/유치원 학부모
- 2차: 학원/동아리 운영자

## 4) MVP 사용자 플로우
1. 카카오 로그인
2. 아이 프로필 생성
3. 아이 얼굴 사진 3~5장 등록
4. 동영상 업로드
5. 서버 처리(등록 얼굴만 비블러, 나머지 블러)
6. 결과 다운로드
7. 결과 파일 24시간 뒤 자동 삭제

## 5) MVP 기능 범위
### Must
- 카카오 로그인
- 아이 얼굴 등록(최대 1명, 3~5장)
- 동영상 업로드/처리/다운로드
- 자동 삭제(24시간)
- 실패/대기/완료 상태 표시

### Should
- 처리 옵션: 속도 우선 / 품질 우선
- 업로드 파일 제한(예: mp4, 최대 30초, 720p)

### Won't (v0.1 제외)
- 실시간 스트리밍 처리
- 다중 아이 프로필 고급 관리
- 공개 무제한 API

## 6) 기술 설계 (현재 코드 기반)
- Detection: MTCNN
- Embedding: ArcFace ONNX (512-d)
- Matching: 코사인 거리
- Processing: OpenCV Mosaic
- Backend: FastAPI
- Worker: 비동기 작업 큐(초기엔 단일 워커)
- Storage: 업로드/결과 파일 임시 저장 + 자동 정리

## 7) 개인정보/보안 원칙 (필수)
- 얼굴 임베딩은 민감정보로 간주한다.
- 원본 얼굴 사진은 임베딩 생성 후 즉시 삭제(기본 정책).
- 동영상 원본/결과는 단기 보관 후 자동 삭제.
- 사용자별 데이터 접근 격리(타 사용자 접근 불가).
- 전송 HTTPS, 저장 암호화, 서명 URL 짧은 만료.
- "내 데이터 즉시 삭제" 기능 제공.

## 8) 성능/비용 운영 정책 (학생팀 버전)
- 비동기 처리만 지원(실시간 X)
- 기본 제한: 30초 / 720p
- 기본 최적화: 프레임 스킵 + 리사이즈
- 동시 처리 수 1~2개 제한
- 사용량 로그로 과금 폭탄 방지

## 9) 성공 지표 (v0.1)
- 처리 성공률 >= 95%
- 평균 처리시간(30초 영상 기준) 목표치 달성
- 주요 실패 케이스 로그 수집 완료
- 외부 사용자(팀 외 3명 이상) 데모 성공

## 10) 4주 실행안
- Week 1: 비디오 업로드/처리 파이프라인 완성
- Week 2: 로그인 + 얼굴 등록 + 임베딩 저장
- Week 3: 결과 다운로드 + 자동 삭제 + 배포
- Week 4: 안정화(에러처리/문서/데모 스크립트)

## 11) 리스크 & 대응
- 오탐/미탐: 확신 낮으면 블러 유지 기본값
- 비용 초과: 길이/해상도/동시성 제한
- 정책 이슈: 동의/고지 문구 및 삭제 기능 우선 구현

## 12) 다음 구현 순서 (바로 시작)
1. `/process-video` 비동기 잡 API 설계
2. 임베딩 저장 스키마(유저ID, 아이ID, 벡터, 모델버전)
3. 자동 삭제 스케줄러(원본/결과)
4. 카카오 로그인 연동
